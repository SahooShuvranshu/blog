## What Problem Was Git Trying to Solve?

Before Git, developers still needed to answer basic questions:

- What changed in the code?
- Who changed it?
- When did it change?
- How do we undo it?

This idea is called **version control**.

Version control means keeping a history of your project so you can:
- Go back to older versions
- Compare changes
- Experiment without fear
- Work with other people safely

Git is the tool that made this simple, fast, and reliable.

---

## What Is Git? (Explained Simply)

Git is a **version control system**.

At its core, Git:
- Tracks changes in files
- Saves those changes as *snapshots* called commits
- Allows branching, merging, and rollback
- Works offline
- Is extremely fast

Think of Git as a **timeline for your code**.

Every time you make a commit, Git takes a photo of your project at that moment.  
You can always go back to any photo.
